Each user is assigned one or more roles. By default there are two roles: anonymous (a user who has not logged in) and authenticated (a user who has signed up and been authorized). Without signing up, users are extremely limited in the types of content they can create. Signed up users also benefit by allowing them to associate content they create with their account and by allowing various permissions to be set for their role within the site.

Users can fine tune some personal configuration settings through their individual my account page. Registered users need to authenticate by supplying either a local username and password or a remote username and password, such as an OpenID.

Authenticated users can be given the additional roles of contributor, editor or maintainer by any maintainers of the site. The person who applied for the site automatically has the maintainer role.

Permissions for user roles (roles are additive, so an authenticated user has the roles cited in addition to the ones cited for the anonymous user, a contributor has the roles cited in addition to the ones cited for the authenticated user, etc.):

Anonymous user:

  • view, search and print all data which are available for the public
  • post comments if comments by anonymous users are allowed on the site

Authenticated user:

  • create forum topics
  • post comments
  • vote on polls
  • subscribe to newsletters

Contributor:

  • create nodes and edit own node content (pages, bibliographic citations, specimen data)
  • upload images
  • set up polls

Editor:

  • edit and administer node content
  • import nodes and bibliographies
  • administer menu
  • administer comments
  • configure group member roles
  • send newsletters
  • administer classification/taxonomy

Maintainer:

  • administer blocks, forums, image galleries, Terms & Conditions, organic groups, panels, panel views, newsletters, site configuration, views
  • create panel pages, newsletters
  • assign user roles
  • access statistics
  • edit webforms and access results
