Scratchpad data can be published in the journal ZooKeys by using the 'Publication' module, which is based on the 'Organic Groups' module. A prototype for this workflow has been developed (Blagoderov et al. 2010). Within this workflow, the prototype of the 'Publication' module on the Scratchpads is functional, but the review process and data exchange between Scratchpads and ZooKeys are not fully automated yet and will be further developed as part of the ViBRANT project. Therefore, before creating a publication, you need to be aware that it may take a while before the whole process is automated. If you do not want to wait for this, you need to ask ZooKeys whether they will accept your Scratchpad publication and if yes, coordinate with them.

The 'Publication' module currently allows you to select data from the biblio, image, profile, and taxon description content types and to integrate these data into the publication.

  • If you want to describe new taxa in your publication you need to add temporary names for these taxa to the classification first in order to link the just mentioned data to these names. These temporary names will be visible in your classification for everybody, so choose names like "Genus sp. 1" or "Genus sp. B" to avoid confusion with 'real' names.
  • To create a publication click on the 'Add a Publication node' icon which appears when you hover over 'Publication' in the Content menu. Enter a 'Short name' and a 'Publication title'. You don't need to change the other settings. Even though 'Published' is checked under the 'Publishing options' section, nobody but group members will be able to view more than the short title of the publication (See Add a new member to a group). Click on the 'Save' button.
  • You are now in the 'Edit' mode which has several different tab sections. Under the 'Metadata' tab you can add data which are typical for all publications, like abstract and keywords. Authors are chosen from the 'Profile' content type. If you don't have a profile for an author you need to click on the 'Create author' button and enter the data you need. Under 'Institution' enter the full address which is to appear in the paper, not just the institution name. Save. To select authors for the publication click on the 'Edit publication authors' button. The available authors are listed at the bottom. Click on an author to select him/her. The name will now appear in the upper left box (note that the title of the author will not appear in the publication). Use the buttons below the box to sort authors or use drag and drop. Click on the 'Save' button. Always click the 'Save' button before you leave a tab section.
  • Click on the 'Taxa' tab. Select a classification, extend the classification by clicking on the plus icons and select the taxa which you want to include in your publication. Save.
  • Click on the 'Default fields' tab. Here you can select the fields from the 'Taxon Description' content type that you want to show in your publication by default. These will usually be the fields you have actually filled in. You need to add 'Taxon descriptions' for each of the taxa for which you want to include descriptions in your publication. We recommend to use the 'Morphology' field for the actual morphological description of the taxon and 'Diagnostic Description' for the diagnoses. Save.
  • Click on the 'Sections' tab. You can see a list of the taxa you selected including a taxon description for each.
    • Click on 'Edit' next to a taxon term to enter the final published name for this taxon in case of new taxa. This is also the place to enter citation data if the taxon has been cited elsewhere, and the type material. Save.
    • Click on 'Edit term fields' next to a taxon description to select the fields from the 'Taxon Description' content type that you want to show only for this taxon. The default fields are preselected.
    • Click on the 'Add section' link below the blue box to add new custom sections like "Introduction" or "Discussion".
    • Drag and drop to re-arrrange the different sections using the cross icons in front of each section. Dropping slightly to the right will create a hierarchical order.
    • Save.
  • Click on the 'Images' tab.
    • To add new images click on 'Create images', enter the respective data, tag it to the classification, and browse for the image. Make sure that under the 'Advanced options' at the bottom, when you click on the 'Access' tab, your publication short title is selected and 'Public' unselected. This will create an image accessibly only to group members until the paper is published.
    • To select the images you want to include in the publication click on 'Edit publication images'. Select an image by clicking on its thumbnail. Click on 'Filter items' to filter for images of certain taxa. The title of the images will appear in the upper box. Drag and drop the titles into the correct order. Save.
    • To add figure captions click on an image thumbnail and enter the caption into the box. Save. Images with captions are marked with a yellow quadrangle in the upper right corner of the thumbnail. Save.
  • Click on the 'Bibliography' section. This work the same way as the image section, but you have more options for filtering for specific references. Make sure to sort you references alphabetically by clicking on the 'Desc.' button before you save.
  • The last tab is the 'Privacy' tab, which allows you to edit your 'Publication' settings. You don't need to change any of the default settings.
  • Click on 'View' to preview your publication and check for errors or missing data. 'Edit' the publication to add or correct data if necessary. The data are transferred to the journal via XML, so it is also useful to click on 'XML' to check for missing data (In most browsers you will not see XML unless you select "View Source"). Especially the bibliographic references can miss some of the rarer fields, so if any fields you need are missing, you need to contact us at scratchpad@nhm.ac.uk. The 'Export' function is not working, yet. So you need to save the XML (right click and 'Save link as') and send it to ZooKeys by email.
  • The 'Make public' button will make all data connected to a publication public, so this button should only be pressed AFTER the printed version of the publication has been published by ZooKeys to avoid confusion about the availability of new names. You will be notified by ZooKeys when this has happened.
