Format of revision

  • Assuming you have a Word document with the final manuscript, you would copy text from this document into the Scratchpad rather than copying from a pdf.
  • If the text in your document is consistantly structured, like a checklist for example or species descriptions, it can also be an option to transfer these structured data into a tab delimited file by replacing certain key characters (e.g. the paragraph sign) with the tab character and then cleaning up the resulting matrix in Excel. There are also special programs, like Golden Gate, that parse data.

Structure of revision on Scratchpad

  • Generally speaking the taxonomy of the revised group should be part of the taxonomic classification on the Scratchpad in order to generate taxon pages with the descriptions, images, specimens and biblio. The general information of the paper goes into pages which can be linked via a menu. You can check out the revision of the genus Milichiella at http://milichiidae.info/content/milichiella-online-revision and a smaller online-only paper at http://milichiidae.info/content/milichiidae-lake-kerkini-region-greece. These are just examples, obviously you can do it differently if you want.

Steps to take

  1. Taxonomy: If you don't have a taxonomic classification on your Scratchpad, create a new ITIS name vocabulary. Either create a file in the correct format and import the nomenclatural data or add the data to the Editor by copy and pasting them from your manuscript. You should add valid/accepted names as well as synonyms, basionyms/original combinations or subsequent combinations to the classification and optimally link them to the bibliography after you imported the literature.
  2. Literature: Import the literature into the Scratchpad or add the citations by hand. This should include the citation of your revision. The biblio of your revision should have the pdf attached if possible, or include a link to the pdf. It should also be tagged with all the taxa covered.
  3. Specimens: Import the localities and specimens (see Manual for advanced training course) or add localities and specimens by hand.
  4. Images: Add the images and tag them with the classification, the biblio of your revision and if possible also the specimens. If you have plates, you can tag them to all the species shown on this plate. However, if there are figures of several species on a plate it is better to separate the figures because otherwise figures will show up on the wrong species pages.
  5. Taxon descriptions: The best content type for descriptions is the Taxon description content type because you can later show these descriptions in EOL if you want. However, the fields are quite general and you might want to have a more anatomised description. If this is the case you have to create a custom content type with the fields you need.
  6. Phylogenetic trees: Add phylogenetic trees and link them to the classification.
  7. General information: General chapters like 'Materials & Methods' or 'Introduction' should be added as pages and linked via a menu for this revision.
  8. Morphological information: General morphological information that is not part of the taxon descriptions can either be added as pages or as a glossary.
  9. Acronyms: Museum acronyms can also be add as a glossary, however, you can only have one glossary. So if you have a morphological glossary, you can add a list of acronyms as a page.
  10. Keys: The best option would be to generate an interactive key and show it on the Scratchpad. Dichotomous keys can be added as pages with the terminal taxa linked to the respective taxon pages. Dichotomous keys are highly formatted, so it is a bit of work to recreate this format on the Scratchpad. If you want to avoid this, you could create a pdf of the key and attach the file to the description of the higher taxon, e.g. the genus if the key is to the species of this genus.
