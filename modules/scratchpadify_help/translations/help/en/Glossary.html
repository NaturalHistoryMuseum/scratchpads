A Scratchpad glossary is basically a vocabulary. The term names of this vocabulary will be highlighted on all normal pages and in all filtered text fields of custom content types. Clicking on a highlighted glossary term will link to the term page and hovering over the term will show the description of the term if this field has been filled in in the taxonomy module. If synonyms have been entered for a term, these will be highlighted, too, and linked to the page of the valid term as well as show the description of the valid term whilst hovering. A glossary term page consists of the term name and (optionally) the description. The general glossary page (.../glossary) shows an alphabetical list of terms either on one page or separated into pages for each letter.

By adding parent terms a glossary can be made hierarchical and this hierarchy can be shown as a menu using the tinytax module. However a hierarchical glossary interferes with the alphabetical sorting on the general glossary page, because only the child terms for one parent term are sorted. If the glossary page is set so that one page is shown for each letter, terms starting with this letter will appear on the correct page, but within this page they will show in hierarchical order, not alphabetical order.

It is possible to connect a custom content type to a glossary in order show more information for the terms and attach images or files. In this case a glossary term page shows the data added to this content type in addition to the term name and the description.

To create a glossary you need to first create the glossary vocabulary. If you want to add more information, you then need to create a new custom content type for your glossary. To set up the glossary you need to contact the site administrator because only administrators have access to the glossary module.
